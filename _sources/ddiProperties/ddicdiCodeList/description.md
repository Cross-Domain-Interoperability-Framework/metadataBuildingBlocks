DDI-CDI CodeList models a list of codes paired with the categories they denote. The root `cdi:CodeList` carries `cdi:has_Code` (the codes themselves), `cdi:has_CodePosition` (their ordering), `cdi:references` to a `cdi:CategorySet` of underlying categories, and `cdi:isDefinedBy` linking to one or more SKOS concepts. A `cdi:LevelStructure` may be attached via `cdi:uses` for hierarchical code lists.

The local `$defs` cover the supporting types DDI-CDI uses to express controlled value sets: `Code`, `CodePosition`, `Category`, `CategoryPosition`, `CategorySet`, `Notation`, plus `ClassificationItem`, `Level`, `LevelStructure`, and `AuthorizationSource` shared with statistical classifications. CodeList is the building block referenced from `ddicdiEnumerationDomain` and indirectly from value domains when a variable's substantive values come from an enumerated list.
