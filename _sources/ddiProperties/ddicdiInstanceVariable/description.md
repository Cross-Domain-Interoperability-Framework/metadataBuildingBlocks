DDI-CDI InstanceVariable describes the use of a represented variable within a particular dataset, binding the conceptual variable to its physical realization. The root `cdi:InstanceVariable` carries physical-level properties `cdi:physicalDataType`, `cdi:platformType`, and `cdi:source`. In the 2026-03 DDI-CDI model the physical layout of a variable's values is described from the mapping side (`PhysicalMapping` → `formats` → `InstanceVariable`), so the InstanceVariable no longer carries outward `has_PhysicalSegmentLayout` / `has_ValueMapping` links — see the `ddicdiPhysicalMapping` BB.

It also carries the conceptual-level properties shared with `ddicdiRepresentedVariable`: `cdi:takesSubstantiveValuesFrom` and `cdi:takesSentinelValuesFrom` (referencing `ddicdiValueDomain`), `cdi:measures` (a local `UnitType`), `cdi:describedUnitOfMeasure`, `cdi:hasIntendedDataType`, `cdi:variableFunction`, and the substantive/sentinel conceptual-domain links. Local `$defs` cover `ConceptSystem`, `SubstantiveConceptualDomain`, `SentinelConceptualDomain`, `UnitType`, `ValueAndConceptDescription`, and `TypedString`. InstanceVariable is the variable-as-realized counterpart to `ddicdiRepresentedVariable` in the CDIF Data Description profile.
