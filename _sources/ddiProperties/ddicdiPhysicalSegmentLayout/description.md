DDI-CDI PhysicalSegmentLayout is an extension point used to describe the physical layout of a data segment — typically a delimited or fixed-width text file, but applicable wherever a physical layout must be characterized. The root `cdi:PhysicalSegmentLayout` carries the W3C tabular-data-model parameters (`cdi:delimiter`, `cdi:encoding`, `cdi:escapeCharacter`, `cdi:hasHeader`, `cdi:headerIsCaseSensitive`, `cdi:commentPrefix`, `cdi:arrayBase`, etc.) along with `cdi:allowsDuplicates` and additional dialect properties.

It is referenced from `ddicdiInstanceVariable` and `ddicdiPresentationalVariable` via `cdi:has_PhysicalSegmentLayout` so that each variable can be associated with the file layout in which its values appear. Within the CDIF Data Description profile it complements the per-column `ValueMapping` carried inside InstanceVariable.
