DDI-CDI EnumerationDomain is a base class acting as an extension point so that any codification — code list, statistical classification, etc. — can be treated as an enumerated value domain. The root `cdi:EnumerationDomain` carries `cdi:references` to a `cdi:CategorySet`, optional `cdi:uses` of a `cdi:LevelStructure` for hierarchical structures, `cdi:isDefinedBy` linking to one or more SKOS concepts, and the standard `cdi:identifier`, `cdi:name`, and `cdi:purpose` properties.

Local `$defs` mirror the supporting types from `ddicdiCodeList` (`Code`, `Category`, `CategorySet`, `ClassificationItem`, `Level`, `LevelStructure`, `Notation`, `AuthorizationSource`). It is referenced by `ddicdiValueDomain` (`cdi:takesValuesFrom`) so a substantive or sentinel value domain can pull its allowed values from any kind of enumerated codification.
