Attributes

NameTypeMultiplicityDescription
definitionstring[1]Natural language statement conveying the meaning of a concept, differentiating it from other concepts. Supports the use of multiple languages and structured text. 'externalDefinition' can't be used if 'definition' is used.
describedUnitOfMeasureConcept ↗[0..1]The unit in which the data values are measured (kg, pound, euro), expressed as a value from a controlled system of entries (i.e., QDT). Supports the provision of an identifier for the entry in the authoritative source (a URI, etc.), and the specific vocabulary.
unitOfMeasureKindConcept ↗[0..1]Kind of unit of measure, so that it may be prone to translation to equivalent UOMs. Example values include "acceleration," "temperature," "salinity", etc. This description exists at the conceptual level, indicating a limitation on the type of representations which may be used for the variable as it is made more concrete.
functionConcept ↗[0..*]cdi:function - immutable characteristic of the variable (geographic designator, weight, temporal designation, ...).
platformTypeConcept ↗[0..1]cdi:platformType - the application/technical system context in which the variable was realized.
sourcestring[0..1]cdi:source - provenance information for this InstanceVariable.
hasIntendedDataTypeConcept ↗[0..1]cdi:hasIntendedDataType - the data type intended for this variable, independent of its physical representation.
simpleUnitOfMeasurestring[0..1]cdif:simpleUnitOfMeasure - plain-string unit of measure (use describedUnitOfMeasure for a controlled-vocabulary unit).
usesConcept ↗[0..*]cdif:uses - concept(s) this variable measures or represents (akin to schema:propertyID).
