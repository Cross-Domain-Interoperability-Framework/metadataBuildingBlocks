Should we use skos:Collection for Codelists?  A skos:Collection has a skos:member property that enables embedding a set of skos:Concepts inside the collection.  ConceptScheme only has a 'hasTopConcept' property, which means you have to navigate all the skos:narrower links to get the whole codelist if there is any hierarchical structure.   In a collection you can just enumerate the concepts in the codelist; there can still be hierarchy, but a Collection doesn't have a topConcept, so its tricky to identify the top of the hierarchy -- a sparql query for concepts that do not have a skos:broader relation....  Or, make the codelist both a skos:Collection and skos:ConceptScheme...
